注文情報統合・記帳支援アプリ (Order Formatter) 開発仕様書
1. 開発目的
AIによって抽出された注文データに対し、人間が「処理担当者」「備考」などの動的な情報を追加し、スプレッドシートの特定セルへ貼り付け可能な1行のテキストデータ（TSV形式）を出力するWebアプリケーションを開発する。
2. 技術スタック
Frontend: React (18+)
Build Tool: Vite
Language: TypeScript
Styling: SCSS (Sass)
State Management: React Hooks (useState, useEffect)
UI Components: モダンで直感的なUI。Lucide Reactなどのアイコンライブラリを推奨。
3. アプリケーション機能要件
A. データ入力・自動反映
AI（ChatGPT等）が画像から抽出した情報を、JSON形式等でペーストして一括反映できる入力エリア、または個別の編集フィールドを持つこと。
自動反映項目（AI抽出分）:
受付日 (YYYY/MM/DD)
お名前
注文番号
購入先（「Amazon」または「楽天市場」のいずれかを選択/出力）
ASIN（Amazonの場合のみ。楽天の場合は「-」）
商品名
個数（総個数：n個セット×m個注文を計算した解）
B. 手動選択・入力項目
届いた日: 日付選択式（DatePicker）。デフォルトは今日。
個数増減: 数値入力。デフォルトは「0」。
処理した人の名前: 以下のプリセットから選択（Dropdown）。
プリセット：[友川、ヒロセ、桒村、椎野、松浦、森本、宮原]
備考: テキスト入力（手打ち）。
C. 出力機能（書き込みボタン）
「書き込み（コピー）」ボタンを押した際、以下の11項目のヘッダー構造に従い、タブ区切り（TSV）でクリップボードにコピーする。
ヘッダー構造（列順）:
受付日
お名前
注文番号
購入先
ASIN
商品名
個数
届いた日
個数増減
処理した人の名前
備考
4. デザイン要件 (SCSS)
レスポンシブ対応。
スプレッドシートへの記帳作業を効率化するため、視認性の高いフォームデザイン。
入力漏れを防ぐためのバリデーション（必須項目の警告等）。
5. モール名に関する制約
購入先（モール名）のリンク化を防ぐため、「Amazon.co.jp」ではなく「Amazon」と表記すること。
楽天市場は「楽天市場」と表記すること。
6. エージェントへの指示
この仕様書に基づき、src/App.tsx、src/App.scss、および必要なコンポーネントを作成してください。
vite を使用した環境構築手順と、依存ライブラリ（sass, lucide-react, date-fns等）のインストールコマンドを提示してください。
初期表示時に「AIからのデータを貼り付けてください」というプレースホルダーを持つテキストエリアを配置し、そこにJSONを貼ると各フォームが埋まるロジックを実装してください。
